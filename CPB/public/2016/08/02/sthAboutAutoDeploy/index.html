基础轮子，hexo自动发射

发表于 2016-08-02 | 分类于 blog | |
| 字数统计
简单点，发射的方式简单点

非常感谢这个博主的文章

但是为了方便以后查阅还是在这里用自己的方式记录下

母鸡（VPS）上的配置：

  • 首先母鸡上有关要有git

  • 原博主开多了个专门的用户，但我觉得没有必要（也许以后的我会认为这是错的）

  • 完成基础轮子 SSH Key登录设定

  • 在母鸡上找个位置初始化git库（这里以 /var/repo/blog.git/ 为例）

    $ sudo mkdir /var/repo
$ cd /var/repo
$ sudo git init --bare blog.git    #bare裸仓库，裸仓库没有工作区，我们不会在裸仓库上进行操作，它只为共享而存在。

  • 配置hooks，现在的我并不明白这个，希望以后的我能比我更清楚它->GitHooks
    Anyway,这里要利用git操作完成后会运行一次的 post-receive

    $ cd /var/repo/blog.git/hooks
$ vim post-receive    #新建这个文件

    然后写入

    #!/bin/sh
git --work-tree={你要部署的目录} --git-dir=/var/repo/blog.git checkout -f

    最后要加上执行权限

    chmod +x post-receive

客户机（本地）的配置：

  • 修改 hexo 目录下的 _config.yml 文件，找到 [deploy] 条目，并修改为：

    deploy:
  type: git
  repo: 用户名@VPS地址:/var/repo/blog.git #SSH端口似乎一定得是默认的22端口？
  branch: master
  • 最后，发射
    hexo clean && hexo generate --deploy
# blog
