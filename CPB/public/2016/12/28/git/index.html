Commiting change

Checking status

Back to privious version(everything)

  • find your commit, and copy the first part of string(i.e: a5f620e of “a5f620e HEAD@{0}: reset: moving to HEAD~1”) aka the commit_id

  • entering git reset --hard 'commit_id'

  • you can also use ‘HEAD’ to represent the present version, ‘HEAD~N’ to represent the privious Nth version( i.e: git reset --hard HEAD~1)

Back to privious version(certain file)

There are 3 different situations in this case.

One:Haven’t added to temporary stage.

  • simply using git checkout -- file, bring this file to what it was when you commited it last time.

Two:Already have added to temporary, but luckily you haven’t commited it yet.

  • using git reset HEAD file, bring you to situation one.

Three:Already commit.

  • Refer “Back to privious version(everything)”
